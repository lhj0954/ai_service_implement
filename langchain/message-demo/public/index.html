AIMessage 속성 이해
SystemMessage + HumanMessage를 전달하면 AIMessage가 반환됩니다.
반환된 AIMessage의 content, tool_calls, usage_metadata, response_metadata 속성을 확인합니다.
AIMessage

ToolMessage — Tool Calling 전체 흐름

        
Human → AIMessage(tool_calls) → ToolMessage(결과) → AIMessage(최종)

          4단계 메시지 흐름을 통해 Tool Calling이 어떻게 동작하는지 확인합니다.

      

      ToolMessage
    

    

      
// 1. Tool 바인딩
const modelWithTools = model.bindTools([multiply, add]);

// 2. 첫 호출 → AIMessage with tool_calls
const aiResponse = await modelWithTools.invoke([new HumanMessage(question)]);

// 3. Tool 실행 결과를 ToolMessage로 래핑
const toolMsg = new ToolMessage({ content: result, tool_call_id: tc.id });

// 4. 전체 이력 포함 재호출 → 최종 AIMessage
const final = await modelWithTools.invoke([humanMsg, aiResponse, toolMsg]);

MessagesPlaceholder — 대화 이력 동적 삽입

        
System → Few-shot → MessagesPlaceholder(이력) → Human(현재 질문)

          질문을 반복할수록 이전 대화 맥락이 프롬프트에 누적됩니다.

      

      MessagesPlaceholder
    

    

      
const messages = [
  new SystemMessage("한국어 선생님 페르소나..."),
  new HumanMessage("자음 탈락이 뭐야?"),  // Few-shot
  new AIMessage("..."),               // Few-shot
  // ↓ MessagesPlaceholder("chat_history") 위치
  ...chatHistory,            // 런타임에 동적 삽입
  new HumanMessage(question),
];


      
      
대화 이력 (MessagesPlaceholder에 삽입됨)

      

        
아직 대화 이력이 없습니다. 질문하면 이력이 쌓입니다.
